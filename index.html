Phase: Identification

The Hacking Cycle

Hacking is a structured process. To defend a system, we must understand the 5 phases of a professional attack.

  • Reconnaissance: Gathering information on the target network.
  • Scanning: Identifying open ports, live hosts, and service versions.
  • Gaining Access: Exploiting vulnerabilities to enter the system.
  • Maintaining Access: Establishing persistence via backdoors.
Phase: Exploitation

Password Cracking

How credentials are bypassed or stolen in modern environments.

  • Dictionary Attacks: Using lists of common passwords against a target.
  • Brute Force: Calculating every possible combination (time-intensive).
  • Rainbow Tables: Using pre-computed hash tables to crack salted passwords faster.
  • Social Engineering: Tricking users into revealing credentials.
Phase: Escalation

Privilege Escalation

Once inside as a guest, the goal is to become the Administrator (Root).

  • Vertical Escalation: Gaining higher-level permissions (Guest -> Admin).
  • Horizontal Escalation: Gaining access to another user's files at the same level.
  • Exploiting Misconfigurations: Leveraging weak file permissions or unpatched kernel vulnerabilities.
Phase: Advanced Exploit

Buffer Overflow

As developers, this is our biggest risk. When a program writes more data to a buffer than it can hold.

  • Memory Corruption: Overwriting the Return Address in the stack.
  • Payload Execution: Redirecting execution flow to malicious shellcode.
  • Defenses: Implementing ASLR, DEP, and safe coding practices (using `strncpy` over `strcpy`).
Phase: Maintenance

Post-Exploitation

What a hacker does after successfully owning the system.

  • Pivoting: Using the hacked system as a jump-box to attack internal servers.
  • Data Exfiltration: Quietly moving sensitive company data out of the network.
  • Clearing Tracks: Scrubbing Event Logs and bash history to hide footprints.
