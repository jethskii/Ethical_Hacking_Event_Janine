Security Architecture

Establishing a "Defense in Depth" strategy. Modern web applications require layered technical controls to protect against complex external threats and internal vulnerabilities.

  • Defense in Depth: Creating multiple layers of security so if one fails, others protect the system.
  • Perimeter Defense: Filtering malicious requests before they reach the server (WAF).
  • Data Sovereignty: Ensuring information is unreadable without authorization (Encryption).

WAF Control

Web application firewalls (WAFs) are the first line of defense at the application layer. They provide visibility into application data at the HTTP Application Layer (Layer 7).

[ FIG 1.2: OSI Reference Model - WAF operates at Layer 7 ]

L7 Application: Direct interaction with software (HTTP/FTP). Where WAF lives.

L6 Presentation: Data translation, encryption, and compression.

L5 Session: Manages connections and conversations between devices.

L4 Transport: End-to-end communication (TCP/UDP). Standard firewalls live here.

L3 Network: Logical physical path determination (IP Routing).

L2 Data Link: Physical addressing (MAC) and frame management.

L1 Physical: Raw bitstream transmission over hardware (Cables/Wi-Fi).

  • HTTP Methods: WAF inspects GET, POST, PUT, and DELETE requests for malicious patterns.
  • Protection: Safeguards against zero-day exploits, bots, and malware injections.
  • Compliance: Crucial for meeting PCI DSS (Payment Card Industry Data Security Standard).
[DETECTED] GET /admin?id=1' OR '1'='1 [ACTION] 403 Forbidden - SQL Injection block.

Data Encryption

Translating plaintext into ciphertext to protect data from being stolen, changed, or compromised.

  • Types: Symmetric (Same key for lock/unlock) and Asymmetric (Public/Private key pair).
  • Data at Rest: Encrypting information stored on hard drives or databases.
  • Data in Transit: Securing information as it travels over a network (e.g., via HTTPS).
$ openssl enc -aes-256-cbc -salt -in plaintext.txt -out secrets.enc [SYSTEM] Decryption key required for access.

Access Roles (RBAC)

Role-Based Access Control authorizes system access based on predefined roles rather than individual identity.

  • Structure: Administrators assign users to roles (e.g., Finance, HR, IT Analyst).
  • Permissions: Each role has a specific set of privileges (Least Privilege Principle).
  • Internal Security: Defends against malicious insiders and negligent employee mistakes.
Example: A Sales rep can see customer accounts but cannot modify the firewall settings.

Authentication (MFA)

Requires two or more distinct types of verification factors to prove a user's identity.

  • Knowledge: Something you Know (Passwords, PINs).
  • Possession: Something you Have (Smartphone, Security Token).
  • Inherence: Something you Are (Fingerprints, Face Recognition).

Transport Layer (HTTPS)

HTTPS is the secure version of HTTP, using TLS (Transport Layer Security) to encrypt communications.

  • Handshake: Asymmetric encryption establishes a secure link between browser and server.
  • Public/Private Keys: Private key stays on the server; Public key is shared to encrypt data.
  • Indicators: Padlock icons in the URL bar verify the site is using a valid SSL/TLS certificate.
