Defense in Depth

Security is not a single product, but a layered architecture. By separating our controls into different zones, we ensure that if one layer is breached, the others remain secure.

01. Web App Firewall (WAF)

Technical Concept: A WAF operates at Layer 7 of the OSI model. Unlike traditional firewalls, it analyzes the content of HTTP requests to identify and block malicious code like SQL injections.
Real-World Example: An e-commerce system uses a WAF to inspect a login form. If it sees ' OR 1=1 -- in the password field, it identifies it as an injection attack and instantly drops the connection.

02. Encrypted Data

Technical Concept: Information Systems protect data in two states: Data at Rest (on the disk) and Data in Transit (moving over the network). Encryption makes the data useless to anyone without the cryptographic key.
Real-World Example: If a hacker steals the database of a hospital, they see the patient names, but the medical history is stored as random characters like @#89!xs.... This protects the organization from HIPAA violations.

03. Role-Based Access Control (RBAC)

Technical Concept: Access is granted based on the "Role" within the system rather than the user's name. This ensures that permissions are tied to job descriptions, which simplifies auditing and security management.
Real-World Example: In a banking portal, a Cashier can process withdrawals, a Branch Manager can override limits, but an IT Support role cannot see the balance of a private customer's account.

04. Multi-Factor Authentication (MFA)

Technical Concept: MFA requires at least two out of three factors: Something you know (Password), Something you have (Security Key), or Something you are (FaceID/Fingerprint).
Real-World Example: A hacker phishes your password for your school account. However, since the school uses MFA, the hacker is blocked when they can't provide the 6-digit code sent to your smartphone.

05. HTTPS / TLS

Technical Concept: HTTPS provides an encrypted tunnel for data. It uses certificates to verify the identity of the server, preventing "Man-in-the-Middle" attacks.
Real-World Example: When you use your credit card at an online store, HTTPS ensures that the data is encrypted before it leaves your computer, so the person sitting next to you on public Wi-Fi cannot steal it.
