Web App Defense

Web Application Security focuses on the tactical defense of Information Systems. As a Software Engineer, you must ensure that the user interface doesn't become a gateway for organizational collapse.

"In an Information System, the web app is the most visible and vulnerable front door."

01. Broken Access Control

IS Focus: Identity & Access Management

Failure to enforce restrictions on what authenticated users are allowed to do. Attackers exploit these flaws to access unauthorized accounts or sensitive files.

  • Bypassing access control checks by modifying the URL.
  • Permitting viewing or editing someone else's account.
  • Elevation of privilege (acting as admin while logged in as user).
Q: How does this affect an Information System?
A: It violates the Confidentiality aspect of the CIA triad, leading to massive data leaks.

02. Injection (SQLi)

IS Focus: Data Integrity & Reliability

SQL Injection occurs when untrusted data is sent to an interpreter as part of a command or query. The attacker’s hostile data can trick the interpreter into executing unintended commands.

Ice Breaker: Can a "name" field delete an entire company's history?
Answer: Yes, if the system doesn't sanitize the input, a hacker can enter a "DROP TABLE" command.

03. Cryptographic Failures

IS Focus: Compliance & Governance

Formerly known as Sensitive Data Exposure. This is the root cause of data privacy violations. It involves the lack of protection for data at rest and in transit.

  • Using old hashing algorithms like MD5.
  • Transmitting sensitive data in clear text (HTTP instead of HTTPS).
  • Poor key management in the Information System.

04. Insecure Design

IS Focus: SDLC Management

A new category for 2026 focusing on risks related to design flaws. If the design is insecure, no amount of perfect coding can save the system.

Q: What phase of the SDLC should security start?
A: The Requirements phase. Security must be "baked in," not "sprinkled on."

05. Supply Chain Risks

IS Focus: Third-Party Risk Management

Modern apps are built with third-party components. If a library you use in your Information System has a vulnerability, you are vulnerable too.

  • Using outdated versions of React, Angular, or Node.js.
  • Lack of an SBOM (Software Bill of Materials).

06. Social Engineering

IS Focus: Human Resource Security

Psychological manipulation of people into performing actions or divulging confidential information. In 2026, AI deepfakes make this harder to detect.

Q: What is the weakest link in any Information System?
A: The human user.

07. Continuity (BCP)

IS Focus: Business Continuity Planning

The "Availability" of the Information System. Security must ensure that the web application remains accessible even during an attack or system failure.

  • Protection against DDoS (Distributed Denial of Service).
  • Implementing robust data backup and recovery cycles.
