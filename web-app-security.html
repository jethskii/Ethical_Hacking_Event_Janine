Defense-in-Depth

Security is not a single product, but a layered architecture. We use the 5 Pillars of Web Defense to ensure system integrity.

01. Web Application Firewall (WAF)

Technical Mechanism: A WAF inspects HTTP requests (GET, POST, PUT, DELETE) at the Application Layer. It protects against zero-day exploits, malicious bots, and patterns found in headers and query strings.

IS Importance: Essential for protecting retailers and banks, meeting PCI DSS compliance, and safeguarding against malware.

02. Data Encryption

Definition: Cryptography translates plaintext into ciphertext using a secret key. It protects data from being stolen or compromised, whether it is at rest (hard drives) or in transit (transferred over a network).
Symmetric vs Asymmetric: Symmetric uses one key; Asymmetric uses a Public key for encryption and a Private key for decryption.

03. RBAC (Access Control)

Definition: Role-based access control (RBAC) authorizes access based on a user’s predefined role. An administrator assigns roles (e.g., Finance, HR, Sales) which carry specific sets of permissions.
Real-World Example: A sales rep can view customer accounts but cannot touch firewall settings. This protects against malicious insiders and negligent employees.

04. MFA (Multifactor Authentication)

The Three Factors:
  • Knowledge: Something you know (Passwords, PINs).
  • Possession: Something you have (Smartphone, Security Token).
  • Inherence: Something you are (Fingerprints, Face ID).
Example: Combining a password (Knowledge) with a fingerprint scan (Inherence) to minimize the risk of unauthorized access.

05. HTTPS (Secure Transport)

Definition: The secure version of HTTP. It uses Transport Layer Security (TLS) to encrypt communications between a browser and a website.
Asymmetric Infrastructure: Uses a Public Key (available to everyone) to encrypt and a Private Key (kept on the web server) to decrypt. Modern browsers flag all non-HTTPS sites as "Not Secure."
