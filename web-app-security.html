Defense Strategy

To secure a modern Information System, we must deploy layers of defense. These 5 core tools work together to ensure the Confidentiality, Integrity, and Availability (CIA) of our web application.

Figure 1.0: The 5 Pillars of Web Application Security

NETWORK_FILTERING

01. WAF (Web Application Firewall)

Definition: A Web Application Firewall (WAF) acts as a gateway that monitors and filters incoming HTTP traffic. It identifies and blocks malicious requests such as SQL Injection and Cross-Site Scripting (XSS) before they reach the web server.
[Visual: Diagram showing Traffic -> WAF -> Web Server]
DATA_INTEGRITY

02. Encrypted Data

Definition: Cryptography transforms readable data into a complex code (ciphertext). In web applications, we focus on "Encryption at Rest"—protecting databases from physical theft—and "Hashing"—storing passwords in an irreversible format.
[Visual: Comparison of Plaintext vs. Hashed Passwords]
ACCESS_MANAGEMENT

03. RBAC (Role-Based Access Control)

Definition: Role-Based Access Control assigns permissions to specific "roles" rather than individual users. For example, an 'Admin' can edit records, while a 'Customer' can only view their own profile. This enforces the Principle of Least Privilege.
[Visual: Organizational Hierarchy showing Admin vs User permissions]
AUTHENTICATION

04. MFA (Multi-Factor Authentication)

Definition: MFA requires users to provide at least two different forms of identification before logging in. Typically: Something you know (password) + Something you have (phone/key) or Something you are (biometrics).
[Visual: Smartphone showing an Authenticator App code]
SECURE_TRANSPORT

05. HTTPS

Definition: HTTPS is the secure version of HTTP. It uses SSL/TLS protocols to encrypt the communication channel between the user and the server. This prevents attackers from "sniffing" or eavesdropping on private data while it travels across the internet.
[Visual: Address bar with the Padlock Icon and 'HTTPS' highlighted]
