Defense-in-Depth

Security is not a single product, but a process of overlapping layers. If one layer fails, the next one should protect the system.

01. WAF (Web App Firewall)

Discussion: A WAF filters malicious HTTP traffic. Unlike a network firewall, it looks inside the "message" sent to the web app to find hidden code like SQL injections.
Real-World Example: An e-commerce site uses a WAF to block a hacker from entering ' OR 1=1 -- into the search bar, which would normally bypass the login.

02. Encrypted Data

Discussion: Encryption protects "Data at Rest." Even if the server is compromised, the files look like gibberish without the key.
Real-World Example: A bank stores your credit card number as $2a$12$KIX.... If a hacker steals the database, they cannot use the credit cards because they don't have the decryption key.

03. RBAC (Access Control)

Discussion: Roles are assigned to jobs, not names. This ensures the Principle of Least Privilege—only giving the access needed for the task.
Real-World Example: In a university portal, a Student can see grades, a Professor can input grades, but a Professor cannot see another professor's salary records.

04. MFA (Authentication)

Discussion: Password security is dead. MFA adds a physical layer (your phone or fingerprint) so that a stolen password isn't enough to enter.
Real-World Example: You log into Instagram. Even with the correct password, it asks for a 6-digit code sent to your phone. This stops a hacker who guessed your password.

05. HTTPS (Transport Layer)

Discussion: HTTPS encrypts the "tunnel" between you and the server. It prevents "Man-in-the-Middle" attacks on public Wi-Fi.
Real-World Example: When you use your bank's website at Starbucks, HTTPS ensures that the person sitting next to you cannot "sniff" your login credentials out of the air.
