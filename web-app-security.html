Security Architecture

Establishing a "Defense in Depth" strategy. Modern web applications require layered technical controls to protect against complex external threats and internal vulnerabilities.

  • Defense in Depth: Creating multiple layers of security so if one fails, others protect the system.
  • Perimeter Defense: Filtering malicious requests before they reach the server (WAF).
  • Data Sovereignty: Ensuring information is unreadable without authorization (Encryption).

WAF Control

Web application firewalls (WAFs) are the first line of defense at the application layer. They monitor, filter, and block malicious HTTP/HTTPS traffic to prevent unauthorized data access.

  • GET/POST Analysis: Inspecting payload verbs for malicious strings.
  • Exploit Mitigation: Designed to counter SQLi, XSS, and dangerous bots.
  • Deployment: Can be network-based, host-based, or cloud-based solutions.
[STATUS] WAF Filtering Active... [DETECTED] Malicious pattern in POST request /login.php [ACTION] 403 Forbidden - Request Dropped.
Speaker Dialogue: "Think of the WAF as the security guard at the gate. While a standard firewall looks at who is coming in, the WAF looks at exactly what they are carrying inside their HTTP packets."

Data Encryption

Encryption translates plaintext into ciphertext, ensuring data remains secure even if storage or transmission channels are compromised.

  • Symmetric Encryption: Single secret key for both encryption and decryption.
  • Asymmetric Encryption: Public key for encryption, Private key held by the owner.
  • Data States: Protecting data 'At Rest' (drives) and 'In Transit' (networks).
$ openssl enc -aes-256-cbc -salt -in data.txt -out data.enc [SYSTEM] Data translated to Ciphertext.

Access Roles (RBAC)

Role-Based Access Control restricts system access to authorized users based on their predefined roles within the organization.

  • Principle of Least Privilege: Users only access what is needed for their job.
  • Role Assignment: Managing permissions through 'Roles' (HR, Finance, IT) rather than individuals.
  • Audit Integrity: Simplifies tracking and prevents 'Permission Creep'.
Speaker Dialogue: "In a bank, a teller can see your balance, but they can't change the bank's firewall settings. That is RBAC in action—restricting horizontal movement."

Authentication (MFA)

Multi-Factor Authentication requires two or more distinct verification types to prove a user's identity.

  • Knowledge: Something you know (Passwords, PINs).
  • Possession: Something you have (Smartphone, Security Keys).
  • Inherence: Something you are (Fingerprints, Face ID).
[LOGIN] Username: Admin [MFA] Enter 6-digit TOTP code... [SUCCESS] Identity Verified.

Transport Layer (HTTPS)

HTTPS provides an encrypted "tunnel" for data transfer using Transport Layer Security (TLS).

  • SSL/TLS Handshake: Negotiating encryption keys between browser and server.
  • Public Key Infrastructure: Asymmetric keys used to verify authenticity.
  • Trust Indicators: Padlock icons and modern browser security flags.
Speaker Dialogue: "Without HTTPS, your sensitive data is traveling in a clear glass box. Anyone on the same Wi-Fi can see your passwords. TLS turns that box into lead."
