Web App Defense

Establishing multi-layered security protocols to ensure Information System integrity. Defensive strategies focus on mitigation, filtering, and rigorous access control.

Seminar Foundations:

  • Establishing a "Defense in Depth" architecture.
  • Protecting the CIA Triad (Confidentiality, Integrity, Availability).
  • Implementing technical controls for compliance (PCI DSS, GDPR).

01. WAF Control

Web Application Firewalls monitor, filter, and block HTTP traffic to and from a web application. They function at Layer 7 of the OSI model.

Technical Parameters:

  • Payload Inspection: Analyzing GET, POST, PUT, and DELETE requests.
  • Exploit Mitigation: Defending against SQL Injection, XSS, and malicious bots.
  • Visibility: Cloud-based vs. Host-based deployment strategies.

02. Data Encryption

Translating plaintext into ciphertext to ensure that even if data is stolen, it remains unreadable without a cryptographic key.

Encryption States:

  • Data at Rest: Encrypting stored assets on hard drives and databases.
  • Data in Transit: Protecting packets during network transfer.
  • Asymmetric Infrastructure: Utilizing public/private key pairs.

03. Access Roles (RBAC)

Granting permissions based on a user's role within an organization. This limits horizontal and vertical privilege escalation.

Governance Benefits:

  • Least Privilege: Users only access resources required for their role.
  • Simplified Audit: Easier to track permissions in large-scale enterprises.
  • Insider Threat Mitigation: Limits damage from negligent or malicious employees.

04. Authentication (MFA)

Moving beyond passwords by requiring two or more verification factors: Knowledge, Possession, and Inherence.

Authentication Factors:

  • Knowledge: Passwords or PIN numbers.
  • Possession: Security tokens or smartphone authenticator apps.
  • Inherence: Biometrics like fingerprints and facial recognition.

05. Transport Layer

HTTPS (Hypertext Transfer Protocol Secure) provides an encrypted tunnel for communication using Transport Layer Security (TLS).

TLS Handshake:

  • Confidentiality: Prevents eavesdropping on public networks.
  • Integrity: Ensures data is not modified during transit.
  • Authenticity: Certificates verify the server's identity to the user.
