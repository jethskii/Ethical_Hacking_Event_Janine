Web Defense Architecture

Establishing multi-layered security protocols to ensure Information System integrity.

01. Web Application Firewall (WAF)

Technical Mechanism: Inspects HTTP requests (GET, POST, etc.) at the Application Layer to block malicious bots and zero-day exploits.

Importance: Essential for PCI DSS compliance and protecting sensitive data in banking and retail.

02. Data Encryption

Technical Concept: Translating plaintext into ciphertext using cryptography. Protects data "at rest" (storage) and "in transit" (network).

Symmetric: Single key for both actions.
Asymmetric: Public key to encrypt, Private key to decrypt.

03. Access Roles (RBAC)

Technical Concept: Role-Based Access Control (RBAC) authorizes access based on predefined job roles.

Example: A Sales Rep can view customer data but cannot modify firewall settings. This limits internal risk.

04. Authentication (MFA)

Multifactor Authentication (MFA): Combines Knowledge (Password), Possession (Smartphone), and Inherence (Biometrics).

Defense: Blocks 99% of phishing attacks by requiring a physical "Second Factor."

05. Transport Layer (HTTPS)

HTTPS/TLS: Encrypts the communication "tunnel" between browser and server using a Public Key Infrastructure.

Visibility: Modern browsers mark non-HTTPS sites as "Not Secure" to protect user trust.
