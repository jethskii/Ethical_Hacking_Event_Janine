01. Broken Access Control

Definition: Broken Access Control occurs when a web application fails to properly enforce restrictions on what users are allowed to do. It allows attackers to bypass authorization and perform tasks as if they were privileged users or other account holders. IS Impact: Threatens Confidentiality and user privacy compliance.

02. SQL Injection (SQLi)

Definition: An injection attack where an attacker executes malicious SQL statements that control a web application's database server. By inserting code into input fields, hackers can view, modify, or delete data. IS Impact: Directly compromises Data Integrity and system reliability.

03. Cryptographic Failures

Definition: Previously known as Sensitive Data Exposure, this refers to the failure to protect sensitive data at rest (on the server) or in transit (over the network) using strong encryption methods. IS Impact: Leads to massive financial penalties and loss of customer trust.

04. Insecure Design

Definition: A category of vulnerabilities that result from flaws in the logic or architecture of the system. It means the system was built without proper security requirements from the start. IS Impact: High recovery costs; security must be part of the entire SDLC.

05. Supply Chain Risks

Definition: Security risks arising from the use of third-party software, libraries, or APIs. If a component you rely on is compromised, your entire Information System becomes vulnerable. IS Impact: Requires Software Bill of Materials (SBOM) for governance.
