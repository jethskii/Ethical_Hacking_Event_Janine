Security Architecture

Establishing a "Defense in Depth" strategy. Modern web applications require layered technical controls to protect against complex external threats and internal vulnerabilities.

  • Defense in Depth: Overlapping security measures so if one fails, others remain.
  • CIA Triad: Maintaining Confidentiality, Integrity, and Availability.
  • Compliance: Meeting regulatory standards like PCI DSS and GDPR.

WAF Control

Web application firewalls (WAFs) monitor, filter, and block malicious data packets. They provide visibility into application data at the HTTP Application Layer (Layer 7).

  • The "Verbs" Analysis: WAF inspects GET (retrieving data), POST (changing state), PUT (updating), and DELETE requests.
  • Payload Inspection: Checks headers, query strings, and bodies for malicious code like SQLi and XSS.
  • PCI DSS Compliance: Essential for online businesses (banks, retail) to protect credit card data.

Data Encryption

The science of Cryptography translates plaintext into unreadable ciphertext using secret keys.

  • At Rest vs. Transit: Protecting data on hard drives (At Rest) vs. data moving across the web (In Transit).
  • Symmetric Encryption: Faster performance, uses the same key for lock and unlock.
  • Asymmetric Encryption: More secure; uses a Public Key to encrypt and a Private Key to decrypt.
"If a hacker steals an encrypted database, they have nothing but gibberish without the Private Key."

Access Roles (RBAC)

Role-Based Access Control authorizes access based on a user’s predefined role rather than their identity.

  • Principle of Least Privilege: A sales rep can see accounts but cannot modify server firewalls.
  • Privilege Management: Admins assign 'Roles' (Finance, HR, IT) with specific permission sets.
  • Threat Defense: Defends against malicious insiders and negligent employees.

Authentication (MFA)

Multifactor Authentication requires two or more distinct categories of credentials to verify identity.

  • Knowledge Factor: Something you Know (Passwords, PINs).
  • Possession Factor: Something you Have (Smartphone, Security Keys).
  • Inherence Factor: Something you Are (Fingerprints, Face ID).

Transport Layer (HTTPS)

HTTPS is the secure version of HTTP, using TLS (Transport Layer Security) to encrypt communications.

  • TLS Protocol: Formerly known as SSL; it secures the connection between browser and server.
  • Asymmetric PKI: Uses a Private key on the server and a Public key available to everyone.
  • Trust Indicators: Browsers flag non-HTTPS sites as "Not Secure" to protect users.
