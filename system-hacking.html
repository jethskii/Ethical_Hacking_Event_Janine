System Hacking

System hacking is the culmination of all previous phases. This is where we go beyond finding a bug and move toward full machine control. In a professional engagement, this proves to the client that their "unhackable" server can, in fact, be owned.

  • The Goal: To bypass OS security controls and access unauthorized data.
  • The Methodology: Exploiting software bugs, service misconfigurations, or weak credentials.
  • The Responsibility: As ethical hackers, we must document every step to ensure the client can reproduce the fix.
Speaker Dialogue: "Ladies and gentlemen, look at this list. System hacking isn't a movie scene—it's a logical chain. If you miss one step in enumeration, you miss your entry point."

Vulnerability Research

Before exploitation, you must know what you are fighting. We use global databases to find "Zero-Days" or "N-Days" (known flaws).

  • CVE (Common Vulnerabilities & Exposures): A dictionary of publicly disclosed cybersecurity vulnerabilities.
  • Exploit-DB: An archive of exploit code and software for use by penetration testers.
  • CVSS Scoring: Understanding the severity (Critical, High, Medium, Low) of a bug.
# searchsploit "Windows Server 2019"
# RESULTS: 12 Exploits Found (3 Remote Code Execution)

Password Cracking

Most breaches occur because of "admin123." We teach students the physics of cracking passwords and why length matters more than complexity.

  • Dictionary Attack: Testing a list of common words (RockYou.txt).
  • Brute Force: Testing every possible combination of characters (Slower but 100% effective).
  • Rainbow Tables: Using pre-computed hashes to crack passwords in seconds.
$ john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt
$ hashcat -m 0 -a 0 hash.txt pass.txt

Privilege Escalation

Gaining access to a system as a "Guest" or "Standard User" is only half the battle. To be a master, you must become ROOT (Linux) or SYSTEM (Windows).

  • Horizontal Escalation: Gaining access to another user's account at the same level.
  • Vertical Escalation: Moving from a restricted user to an administrator.
  • Kernel Exploits: Attacking the core of the OS to bypass all security.

Buffer Overflows (BoF)

This is the "Holy Grail" of technical hacking. By sending more data to a program than it can handle, we overwrite the memory and force the CPU to execute our own code.

  • Fuzzing: Finding the exact "crash point" of an application.
  • Instruction Pointer (EIP): Gaining control of the pointer that tells the CPU what to do next.
  • Shellcode: The custom payload injected into the overflow to spawn a reverse shell.
Speaker Dialogue: "This is where the math meets the machine. If you can control the EIP, you control the system. No password needed."

Post-Exploitation

What happens after you get in? You must establish a permanent presence and harvest all the intelligence you can find.

  • Data Exfiltration: Stealthily moving sensitive files out of the network.
  • Backdoors: Installing "Netcat" listeners or custom service tasks for persistence.
  • Pivoting: Using your current victim to attack other hidden servers inside the network.

Active Directory Attacks

In 90% of corporate environments, the network is controlled by Active Directory (AD). If you own the Domain Controller, you own the entire company.

  • Pass-the-Hash: Using a user's hash to log in without ever knowing their password.
  • Kerberoasting: Attacking service accounts to gain high-level credentials.
  • Golden Tickets: Creating a permanent "Master Key" to the network that never expires.
# mimikatz "privilege::debug" "lsadump::sam"
# Hash Captured: Admin:500:aad3b435b51404eea...
