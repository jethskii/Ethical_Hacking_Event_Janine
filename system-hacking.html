The Ethical Mindset

To defeat a hacker, you must think like one—but act with integrity. Ethical hacking is the authorized process of bypassing system security to identify gaps.

"Security is not a product, but a process." - Bruce Schneier

Learning Objectives:

  • Understand the Cyber Kill Chain methodology.
  • Differentiate between White, Gray, and Black Hat hacking.
  • Master the technical legalities of a "Rules of Engagement" (RoE) document.

01. Reconnaissance (OSINT)

Information is the most valuable currency. In this phase, we map the "Digital Surface Area" of the target organization.

Key Techniques for Students:

  • Passive OSINT: Using theHarvester to scrape emails and Shodan.io to find vulnerable IoT devices without sending a single packet to the target.
  • DNS Enumeration: Identifying subdomains (e.g., dev.target.com, vpn.target.com) that are often less secure than the main site.
  • Google Dorking: Using advanced search queries like filetype:pdf "confidential" to find leaked documents.
$ whois target-domain.com
$ dig target-domain.com MX
$ site:linkedin.com/in/ "Target Company"

02. Scanning & Enumeration

Now we "touch" the target. We are looking for open ports, specific software versions, and misconfigured services.

Teaching Tools:

  • Nmap (Network Mapper): The industry standard. We teach students to use -sV for version detection and --script vuln to find known bugs automatically.
  • Banner Grabbing: Connecting to a port to see what "banner" the software returns (e.g., "Apache 2.4.49").
  • Vulnerability Scanning: Using professional tools like Nessus or OpenVAS to generate a list of high-risk flaws.
# nmap -sS -A -T4 192.168.1.1/24
# Starting Nmap... Found Port 80 (HTTP), Port 445 (SMB)

03. Gaining Access (Exploitation)

The "Point of Entry." This is where a vulnerability is turned into a working access point.

Core Concepts:

  • Server-Side: Exploiting an unpatched service (like the famous EternalBlue exploit).
  • Client-Side: Tricking a user into running a payload via Phishing or USB-drops.
  • Web-Based: Using SQL Injection to steal passwords or Cross-Site Scripting (XSS) to hijack browser sessions.
Instructor Note: Emphasize that Metasploit is a powerful framework, but students must understand the underlying manual exploits to be true experts.

04. Maintaining Access

Once inside, a hacker wants to ensure they can get back in even if the computer restarts or the password is changed.

Technical Methods:

  • Persistence: Creating hidden 'Cron' jobs in Linux or Registry keys in Windows that execute a backdoor on startup.
  • Privilege Escalation: Moving from a "Guest" account to "System/Root" by exploiting kernel vulnerabilities.
  • Pivoting: Using the first hacked computer as a "jump box" to attack other computers deeper inside the internal network.

05. Covering Tracks

A professional operation leaves no trace. This protects the "persistence" and prevents the Blue Team (Defenders) from knowing they were compromised.

Invisibility Tactics:

  • Log Cleaning: Carefully removing only the specific lines in the System Log that show the hacker's IP address.
  • Timestomping: Changing the "Modified" and "Accessed" dates of malware to match old system files (e.g., set to 2012).
  • Alternate Data Streams: Hiding malicious code inside the metadata of a harmless text file.

06. Reporting & Remediation

The most important part for an Ethical Hacker. We don't just find problems; we provide the cure.

The Professional Report:

  • Executive Summary: Explaining the risks to the CEO in non-technical language.
  • Detailed Findings: A step-by-step guide on how the exploit was achieved so the IT team can replicate it.
  • Remediation Plan: Specific advice (e.g., "Apply Patch MS17-010" or "Disable Port 21").
